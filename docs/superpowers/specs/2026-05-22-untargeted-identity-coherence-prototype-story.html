5. v0.4 改法

V0.4 把鑑別力從 RT gate 移到三個地方：seed coherence gate + specificity context、 independent trace identity checks；background recurrence 只作 audit，不作 identity gate。positive controls 只作 validation yardstick。

8RAW threshold 明定 seed sample count：`seed + at least 2 non-seed`。 避免 3 samples 的嚴格度不清楚。
85RAW threshold 不要沿用 3 samples；先定 count + fraction policy。 3/8 和 3/85 意義差太多。
Positive controls 指定 targeted ISTD / stable-like rows 或 benchmark artifact；不得參與 promotion。 沒有 yardstick，8RAW review 會變成肉眼判斷；但 control label 不能替 row 建立 identity。

8. 驗收 gates

這版不把「看起來合理」當 Go 訊號。第一次 8RAW 是 mechanics learning loop； background / blank / QC validation 必須排在 identity prototype 之後，不能先變成 row-level gate。

必須有的 yardsticks

positive ISTD / stable controls 測 sensitivity；identity decoy 測 false identity specificity。blank / QC / background controls 只能作下游 audit handoff。

必須擋的 failure

RT-only、tier-3-only、單一 tier-1/2 cell + width fallback、request/candidate identity mismatch、identity decoy promoted、forbidden evidence used 都不能成為 identity Go。background-like 只能是 audit flag。

9. 輸出 scope

V0.4 不再一開始凍結大型 detail TSV schema。先凍結最小 review surface。Controls input manifest 叫 `identity_coherence_controls_manifest.yml/.tsv`；frozen output 則是：

untargeted_identity_coherence_cell_evidence.tsv
untargeted_identity_coherence_controls.tsv
untargeted_identity_coherence_summary.md

requests 是 seed/request-level audit surface；cell_evidence 是必要的，因為 per-sample tiered basis、numeric trace metrics、tier-3-only fallback、 background recurrence 只保留 audit flags；final matrix filtering、area correction 與統計解讀留給下游。 candidates/groups 等較大的 detail tables 可以先作為 exploratory artifacts 輸出，等 8RAW 跑過後再凍結。

Implementation gate： 在 config、positive controls、shape metric、process-mode payload 與 evidence-firewall fixture 被 review 接受前，不建議開始寫 identity prototype code；background/QC audit 若要擴充，另開下游 spec。
