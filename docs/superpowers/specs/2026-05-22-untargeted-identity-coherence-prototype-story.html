1. 為什麼現在要停 Backfill

我們最近一直卡在 Backfill，是因為 Backfill 被迫回答它不該回答的問題： 這個 row identity 到底該不該存在。

核心判斷： Backfill 是補值，不是建立 untargeted identity。若前面的 identity 不穩， Backfill 只會把弱 row 放大成 rescue-heavy row，然後我們又回頭替 Backfill 加 gate。

5. Medium seed 的定義

我們先不用 high evidence score。V0.2 的 seed 門檻是中等：fragment evidence 合格， sample-local MS1 owner 可量化，且 peak 欄位完整。

條件V0.2 決策原因
neutral loss / product / observed lossrequired這是 tagged untargeted identity 的最低 fragment evidence。
sample-local MS1 owner 可量化required沒有 area / apex / boundaries 就只是 event，不是可用 seed。
high evidence scorenot required先避免過早把 rare 或中等強度 feature 排除。
ambiguous / duplicate loserexcluded不能用衝突 evidence 建立 primary identity。

6. Coherence scan 是 identity promotion，不是 Backfill

一個 medium seed 先形成 Review candidate。它要進 Primary Matrix，必須通過 seed-centered coherence scan，且至少有 3 個 coherent samples。 

initial_rt_min = seed_rt_min - max_rt_sec / 60.0
initial_rt_max = seed_rt_min + max_rt_sec / 60.0
center guard:  abs(center_rt_min - seed_rt_min) * 60 <= max_center_drift_sec
accept:        abs(apex_rt_min - center_rt_min) * 60 <= preferred_rt_sec

V0.2 baseline:
max_rt_sec       = 180 sec
preferred_rt_sec = 60 sec
max_center_drift_sec = 60 sec
min coherent samples = 3
V0.2 先簡化： coherence 只看 RT + peak completeness。normalized shape、neighbor interference、 scan support、area distribution 先保留成下一階段 scoring model。

7. MVP scope：先不做 production merge / split

V0.2 先用 deterministic primary seed 做 coherence diagnostic。 Secondary / overflow seed 會被完整記錄，但 production graph merge/split 留到 Phase 2。

10. Go / No-Go / Pivot

Prototype 的目的不是多產幾張 TSV，而是跑完 8RAW 後能決定下一步。

8RAW 觀察決策
