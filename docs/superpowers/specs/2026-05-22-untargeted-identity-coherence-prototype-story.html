5. v0.3 改法

V0.3 把鑑別力從 RT gate 移到三個地方：seed coherence gate + specificity context、 independent trace identity checks、background guard；positive controls 只作 validation yardstick。

7. 待你決策的點

問題v0.3 建議為什麼重要
seed specificity 是否要硬 gate V0.3 先不把 evidence_tier A/B 當 gate；若 8RAW 顯示弱 specificity context 仍會 would-primary，再定 diagnostic-owned gate。 避免把 discovery review ranking 誤當 identity specificity。
non-RT identity basis 分級 tier 1 diagnostic NL support，tier 2 shape similarity，tier 3 prototype-width fallback；tier 3 不單獨支撐 would-primary。 避免把普通 MS/MS fragment confirmation 或鬆散 OR gate 誤寫成 identity promotion。
8RAW threshold 明定 seed sample count：`seed + at least 2 non-seed`。 避免 3 samples 的嚴格度不清楚。
85RAW threshold 不要沿用 3 samples；先定 count + fraction policy。 3/8 和 3/85 意義差太多。
Positive controls 指定 targeted ISTD / stable-like rows 或 benchmark artifact；不得參與 promotion。 沒有 yardstick，8RAW review 會變成肉眼判斷；但 control label 不能替 row 建立 identity。
