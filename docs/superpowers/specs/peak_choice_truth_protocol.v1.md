# Peak-Choice Truth Set / Lockbox v1

Status: `production_candidate` evidence-acquisition contract, not product
write authority.

## Plain-Language Contract

This lockbox exists because broad Backfill was parked for lack of independent
peak-choice and area truth. The lockbox does not try to make more rows writable.
It creates a small, reproducible set of cases that humans can label in a
structured way so future automation has truth-like evidence instead of more
heuristics.

Each case asks two separate questions:

1. Is the candidate the correct peak for this sample/family?
2. If the peak choice is acceptable, is the candidate area acceptable?

Reviewer labels are stored separately from product authority. A label may later
support a new goal, but it cannot write to the matrix, switch selected peaks,
change selected areas, change counted detection, or feed ProductWriter by
itself.

## Included Strata

The v1 sampling manifest intentionally covers both positive and negative
contexts:

- approved 511 write-ready controls;
- 3015 trace-matched unresolved rows, split across high-signal dirty,
  low-height, apex-delta, and shape/width/scan risk contexts;
- 1087 missing-overlay rows, as evidence-gap controls;
- failed heldout-oracle rows from already-recorded negative evidence;
- targeted manual wrong-peak / no-peak fixtures.

`quality_blockers` and historical oracle failures are used only to stratify
review coverage. They are not writer predicates and must not be converted into a
new Backfill rule.

## Independence Rules

- The current round-trip oracle is not peak-choice truth and not area truth.
- ISTD evidence may be a limited anchor, but it is not analyte peak-choice or
  area truth.
- Each case requires two independent reviewer labels before agreement metrics
  can be computed.
- Labels must distinguish `correct_peak`, `wrong_peak`, `no_peak`, and
  `unresolved`.
- Area labels must be separate from peak-choice labels.
- Disagreement is evidence; do not hide it by averaging or forcing consensus.

These artifacts are generated by:

```powershell
$env:UV_CACHE_DIR='.uv-cache'; uv run python scripts/build_peak_choice_truth_lockbox.py
```

## Exit Rule

Promote a future automation goal only after the lockbox has enough independent
labels, agreement is computed by family split, and a later goal defines a
separate authority manifest plus expected-diff gate. If labels show unresolved
identity, wrong-family, wrong-peak, or unstable area behavior, keep broad
Backfill parked and use the labels to improve review/evidence recovery instead.
