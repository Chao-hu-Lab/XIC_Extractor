# Backfill Expansion Clean-Target Full-Chain Replay v1

Status: `diagnostic_clean_target_full_chain_held`.

This artifact answers one narrow question: after peak-mode decomposition removes
off-target cells and boundary-bridged target cells, how many clean target-mode
candidates pass when the existing full-chain map is projected through the
selective source-family shift-aware gate and matching MS1 authority sidecar?

It does not read RAW files, change the default matrix, or grant ProductWriter
authority.

## Result

The split decision packet has 18 `family + sample_subtype` groups:

- 112 clean target-mode candidates;
- 37 boundary-review target-mode cells excluded from this replay;
- 29 off-target hold/remap cells excluded from this replay;
- 0 missing or unclassified cells.

Existing full-chain replay, kept as a compatibility baseline:

- 51/112 pass the existing full evidence chain;
- 61/112 remain held;
- full-chain complete: `false`.

Held blockers:

- `shift_aware_gate_blocked_shift_aware_same_pattern_not_supported`: 47 cells;
- `own_max_metric_below_threshold`: 12 cells;
- `own_max_metric_missing`: 2 cells.

Selective projection result:

- 84/112 pass selective source-family evidence and selective MS1 authority;
- 28/112 remain held;
- projected full-chain complete: `false`.

Selective held blockers:

- `selective_own_max_metric_below_threshold`: 21 cells;
- `selective_source_family_shift_attention_only`: 2 cells;
- `selective_source_family_shift_not_same_hypothesis`: 5 cells.

The old 47-cell whole-family shift-aware blocker is no longer the dominant
method blocker for this clean-target subset: 33/47 are recovered by selective
source-family evidence and a selective MS1 authority sidecar.

## Interpretation

Peak-mode cleanup plus selective source-family evidence turns the review
surface into a smaller and cleaner product question. The obvious whole-family
false blockers are handled by the selective MS1 authority sidecar; the remaining
28 cells are real method holds, not rows that need manual review just because
the old family gate was blunt.

The 84 projected passing cells can inform a later expected-diff authority
design, but they are not a write allowlist. The 28 held cells need own-max,
source-family shift, or hold/remap resolution before any authority change.
