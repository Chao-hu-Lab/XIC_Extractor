看圖前，先分清這三件事

讀 overlay 不是背名詞，是分層判斷。一張圖上的東西分三層；每層只能判斷它那層的事，不能跨層下結論。

Layer 1 · 你看到什麼 Signal

MS1 圖上的 peak、trace、RT window、orange seed/context trace。這是「現象」，不是結論。

能判斷：這個 RT 有沒有 MS1 訊號、形狀像不像。
不能判斷：有 signal ≠ 有 identity evidence，更 ≠ 可以 write。
Layer 2 · 這是誰、可不可信 Identity evidence

CID/MS2 tag evidence、source / successor、provenance（state、expected diff）。這層決定 row identity 站不站得住。

能判斷：哪個 row identity 比較可信、provenance 對不對得上。
不能判斷：identity 可信 ≠ 已取得寫入授權。
Layer 3 · 最後能做什麼 Authority decision

write / preserve / review / block，來自 authority gate，不是來自圖。

能判斷：人工 review 傾向支持 / 保留 / 退回 / 阻擋目前 decision 的哪個方向。
不能判斷：圖不改任何 matrix cell、也拿不到 ProductWriter authority。

兩個 review lens ＋ 一條 firewall

Backfill 與 Discovery 是你「用來讀圖」的兩個 lens；Authority 不是 lens，是擋誤升級的 firewall。點下面可淡化其他卡。

Lens 1 · Backfill overlay 在問

既有 row/sample decision 周邊的 MS1 context 是否合理。

  • 看 expected RT window 與 trace 是否對得上。
  • 看 orange seed/context trace 是否和目前 trace 對得上，還是比較像旁邊干擾。
  • 看 decision strip，而不是把 orange trace 當成 write reason。
允許的結論

支持 write/review/block 的人工判讀；不新增 authority。

Lens 2 · Discovery differential 在問

兩個 row identity 誰比較像正確解釋。

  • source/successor m/z 不需要相同，因為它們是兩個 hypothesis。
  • 比較同一組 samples 的 same-sample support pattern，而不是拿單一 m/z 峰型比高低。
  • Gaussian15 smoothing 只讓圖更好讀，不是產品授權規則。
允許的結論

支持 identity migration 或 hold/reject；仍是 review evidence。

Firewall · Authority 在擋（不是一種 overlay）

把「看起來合理」誤升級成 matrix write。

  • Candidate is not a matrix write。
  • MS1 trace 不是 NL tag coverage。
  • ProductWriter authority 需要獨立 activation/export contract。
允許的結論

這張圖可以說服 reviewer，但不能自己修改 default matrix。

你正在審的 XIC 訊號。
orange seed/context trace
真實 Backfill overlay 的 detected seed/context trace；是 MS1 context，不是 authority。
other context trace
旁邊的 QC/context trace；用來判斷是否只是干擾。

該訊號預期出現的 RT 範圍。
family center
同一 family 的參考中心；不是 write decision。

Backfill lens：看 decision context

Backfill overlay 不在問「有沒有任何峰」。它在問目前 row/sample 的 write、preserve、review 或 block decision 是否有合理 MS1 context。 下面 1–3 是主線第 2–3 步（確認 provenance → 用這個 lens 看 signal）在 Backfill 裡的展開。 hover、focus 或點各步驟可高亮對應圖塊；不靠互動也能照文字讀。

Discovery lens：比 row identity

這張圖的重點是「哪個 row identity 更能解釋同一組 samples」。 source/successor m/z 不需要相同；Gaussian15 visual smoothing 只協助讀圖。 下面 1–4 是主線第 2–3 步（確認 row identity/provenance → 用這個 lens 看 pattern）在 Discovery 裡的展開。 hover、focus 或點各步驟可高亮對應圖塊；不靠互動也能照文字讀。

四個最容易看錯的地方

兩個 Backfill、兩個 Discovery。這一段比圖更重要——直接對應目前 Gallery 最讓人失控的原因。

Backfill 誤讀

有 orange context trace ≠ 該 backfill / write

orange seed/context trace 是 MS1 對照訊號，不是 write reason，也不是 tag evidence。 看到 MS1 訊號不代表有 tag evidence；沒有 tag evidence 也不代表 MS1 沒訊號——兩者是不同證據層。

Backfill 誤讀

preserve 是 no-write；context trace 可能只是干擾

preserve ＝ 已有 detected baseline、Backfill 不應覆蓋，是 no-write decision，不是 rescued / write。 而 orange / nearby context trace 只是 context，可能只是干擾，不等於支持目前 row。

Discovery 誤讀

source/successor 不是同一個 m/z 的重畫

Discovery differential 比的是兩個 row identity。m/z 不同是正常的； 你要看哪個 identity story 比較一致，而不是拿同 m/z 疊圖比高低。

Discovery 誤讀

candidate 不等於 matrix row

successor candidate 可以進 review queue，但不是 matrix row、也不是 matrix cell， 不能直接改 ProductWriter authority。

最後用這種語言下結論

結論必須帶上 authority 邊界，而且 Backfill 和 Discovery 用不同的結論模板——Backfill 不是 mapping，別套 accept/reject mapping 的話。

B

Backfill lens 結論

模板：MS1 context supports / does not support the existing decision.

supports：RT window 對得上、orange context trace 不只是干擾 → MS1 context 支持維持既有 decision。
does not support：context 不足或矛盾 → 不支持既有 decision，送 review。

Backfill 不是 mapping、不寫 matrix。write / preserve / review / block 仍由 authority gate 決定。

D

Discovery lens 結論

模板：supports / holds / rejects identity migration.

supports：successor 落在 expected RT window ＋ same-sample pattern 一致 ＋ provenance 對得上 → 支持 migration 進下一個 gate。
holds：至少一層有缺口 → 需要更多 evidence，先不決定。
rejects：successor 偏離 expected window、或 pattern 反向、或 provenance 衝突 → 不支持 migration。

Backfill 對照：支持 vs 不支持既有 decision context（示意）

重點是 MS1 trace 是否落在 expected RT window、orange seed/context trace 是否與目前 trace 一致，不是具體數值。

✓ 支持既有 decision context
  • MS1 trace 落在 expected RT window 內。
  • orange seed/context trace 與目前 trace 一致，不只是零星干擾。
✕ 不支持既有 decision context
  • expected RT window 內幾乎沒有 MS1 trace。
  • orange seed/context trace 偏在窗外或和目前 trace 不一致，比較像干擾。

Discovery 對照：支持 vs 不支持 migration（示意）

重點是 successor 是否落在自己的 expected RT window、same-sample pattern 是否同方向，不是具體數值。

✓ 支持 migration
  • successor 峰落在自己的 expected RT window 內。
  • same-sample pattern 同方向、趨勢一致。
  • provenance（source/successor state、expected diff）對得上。
✕ 不支持 migration
  • successor 峰偏離自己的 expected RT window。
  • same-sample pattern 上下亂跳、沒有一致方向。
圖片可以支持 人工 review、candidate triage、row identity plausibility、下一個 product gate 的討論。
