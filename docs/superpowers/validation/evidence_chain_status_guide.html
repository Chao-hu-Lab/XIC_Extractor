Evidence reading guide · 讀證據前先分層

這頁只回答一件事：每一種證據在看什麼，以及它不能推論到哪裡。

從「在 XIC / MS2 裡看到一個訊號」到「把一個數字寫進對外的定量矩陣」， 中間隔著好幾道 QC 與授權關卡。這頁把這條路上的每一層放進同一套語彙—— 讓熟悉 LC/MS-MS、但不熟悉這套程式模型的人，也能知道眼前證據屬於哪一層， 又不能往上多推一步。

01

The reading model

看到證據，不等於可以寫進公開矩陣

這套系統裡，任何一個判斷都會走過三層。大部分誤讀，都來自把這三層壓成一層—— 看到訊號就以為可以寫、有身分就以為每個樣本都成立。先把三層分清楚，後面的表才看得懂。

質譜語境下可以這樣讀：看到峰或 tag ≠ 確認它屬於該 feature ≠ 寫進交付矩陣。 前者是觀察，後兩者分別是歸屬判定與輸出授權。
E · Evidence 證據

你「看到」了什麼

一個 MS1 峰、一個 MS2 / 中性損失（neutral loss, NL）標籤、一段 XIC trace、一個 RT 吻合。 這一層只回答「有沒有東西可看」，它本身永遠不是「可以寫」的許可。

G · Gate 關卡

把證據變成一個決定

判定關卡把證據裁決成 採用 / 複查 / 暫緩 / 擋下。 就算證據是真的，關卡也可能擋下它——例如這個峰的歸屬不明。

W · Writer 書寫權

才能改動「公開矩陣」

只有登記過的書寫範圍（registered scope），加上預期差異清單（expected-diff，預先聲明會改哪些格） 與來源追溯（provenance，每個值都追得到出處），才能改動對外的 matrix / workbook / CSV。

跟著一個 MS1 峰走一遍
  1. 看到（E）：在某個樣本的預期 RT 窗看到一個 MS1 峰，有面積、有 scan 支撐。此刻只證明「這裡有訊號」。
  2. 這是誰的峰？（歸屬）：它真的屬於我要找的那個 hypothesis 嗎，還是隔壁化合物的訊號？歸屬不明（ambiguous owner）就要先擋下。
  3. 裁決（G）：證據與歸屬夠不夠 → 採用 / 複查 / 暫緩 / 擋下。峰是真的，仍可能因為歸屬不清而停在「複查」。
  4. 書寫權（W）：要把這一格的數字寫進公開矩陣，還得登記範圍、預期差異、來源追溯三者齊備；少一個，就只能停在證據層。
02

Evidence-chain anatomy

把三層展開：一個證據要走過這六步，才可能碰到公開輸出

上一節的三層，實際上拆得更細。同一個峰、同一列、同一格，只要還沒通過後面的關卡， 它就仍然「只是證據」。下面這條漏斗，從上到下能通過的越來越少。

最常見的跳步，是把「看得到訊號」直接當成「可以寫矩陣」。 中間至少還有身分、單一樣本的歸屬、面積來源、預期差異、來源追溯。
1
看到訊號 · signal / tag / trace
2
支持哪個身分 · row / feature / 單一樣本
3
書寫權 · 登記範圍 + 預期差異 + 來源追溯
1 · Observation 看到

先看有沒有可評估的證據

這層包含 MS1 trace、MS2 / NL 標籤、HCD product ion、RT prior、overlay、scan 支撐。 它只回答「有沒有可看的訊號或標籤」。

2 · Hypothesis 支持誰

它支持哪個 row / feature / 樣本格

同一段訊號，可能只是整列的脈絡，也可能是「這個樣本這一格」的證據。 這層要避免把整個 family 的證據，硬投影成每個樣本都成立。

3 · Ownership 歸屬

這個峰真的屬於這個 hypothesis 嗎

歸屬（owner）、峰主取最大（own-max）、跨樣本連線（edge）、歸屬不明（ambiguity）、 位移感知（shift-aware）都在問這件事。它們不是裝飾，也不會自動變成「可以寫」的理由。

4 · Integration 面積

確認面積 / 數值從哪裡來

IntegrationResult、邊界、apex、baseline、峰形面積（morphology area） 決定候選的定量值；但「值的來源」本身仍不是書寫權。

5 · Gate 裁決

把採用 / 複查 / 暫緩 / 擋下講清楚

關卡把證據轉成決策狀態。複查證據、診斷證據、候選證據都可以停在這裡， 不一定要進公開輸出。

6 · Writer 書寫

最後才輪到公開輸出 writer

只有登記範圍、預期差異、來源追溯、schema / 測試全部成立， 證據才可能經由 ProductWriter 影響對外的 matrix、workbook、CSV 或計數偵測（counted detection）。

03

Evidence providers

主要的證據來源，各自只回答一個問題

重點不是「哪個證據最強」，而是每個來源該停在哪一層、什麼時候它仍然只是診斷 / 複查證據。 下表第一欄保留 codebase 裡的正式名稱，方便對照原始碼與 source 文件。

證據來源（正式名稱） 它回答的問題 角色 能推論 / 不能推論
CommonEvidence / decision semantics 把 targeted、Discovery、對齊後格子的 MS1 / MS2 / RT / scan 訊號，翻成可以互相比較的「決定語意」（哪些算支持、哪些是疑慮）。 shared semantics 能解釋一格為什麼被判為 採用 / 複查 / 不計數 / 排除 / 模糊； 但「語意標籤」本身不是書寫範圍。
MS1 trace / morphology / area / boundary 預期窗裡有沒有可評估的 MS1 峰？選到的 apex、邊界、面積、baseline、scan 支撐完不完整？ value evidence 可以給出定量值的來源，也可以是擋下 / 複查的理由。 但「看到峰」或「有面積」都還不等於可以新增一格。
CID-NL / MS2 / HCD / product ion 中性損失（neutral loss, NL）、MS2 產物離子 trace、HCD product ion，支不支持某個身分、Discovery 候選或校正脈絡？ identity evidence 這是「身分」證據，不是書寫權。要寫進公開輸出， 仍需 manifest、預期差異、來源追溯與登記過的範圍。
RT / iRT / drift / paired ISTD 這個 hypothesis 的 RT，跟 RT prior、配對內標（paired ISTD）、漂移模型或鄰近區域相不相符？ context evidence RT 吻合可以加分，RT 不符可以擋下 / 複查； 但「光是 RT 吻合」建立不了書寫權。
Targeted product projection / review actions targeted 輸出裡的 product_state、counted_detection、review_state，是由 projection 決定，還是還在看舊的 Confidence / NL / score？ projection authority targeted 這條線以 Product State / Counted Detection 為計數依據； 會改動結果的 review 動作，需要先核准的預期差異。
Discovery evidence 非目標式（untargeted）的種子，有沒有 MS1 峰、種子事件、嚴格中性損失、產物離子強度、RT 對齊、family 脈絡？ candidate identity 產生的是「候選身分」與啟用證據，它本身不是預設矩陣的書寫者； 不能把候選直接當成矩陣的一列。
Alignment owner / edge / ambiguity 單一樣本的 MS1 峰主是誰？峰主能不能跨樣本連線？歸屬不明、重複、身分衝突該不該擋？ ownership boundary 它回答「這個峰是不是真的屬於這個 hypothesis」，不是只判斷「trace 在不在」。 歸屬不明在解決前就是 blocker。
Backfill evidence projection / authority sidecars 一個原本空白 / 搶救回來的格，有沒有 MS1 樣態、QC 參照、RT 漂移、候選 MS2，以及可追溯的授權旁路檔（sidecar）？ promotion input 這些 sidecar 是授權與追溯檔，不是原始 MS1/MS2 訊號。沒有「已授權的範圍 / 來源 / hash / 追溯」， 這一列仍只是複查或診斷證據。
Quant matrix expected-diff / provenance 對外的預設矩陣，會改哪些格、寫什麼值、來源 hash 與追溯對不對得上？ writer gate 這是最後一道關卡：登記範圍、manifest、預期差異、每格追溯、 未使用差異檢查，必須同時成立。
Review / lockbox / overlay / human guide 人或模型在複查時，怎麼看 overlay、lockbox、trace recovery、challenge queue 與這份教學頁？ review support 可以支援未來的判定關卡；但除非被啟用合約正式採用， 否則預設「fail closed」（不放行），不直接授權公開輸出 writer。
04

Evidence lenses by lane

不同 lane（工作線）看的證據焦點不同，但都走同一道公開輸出 writer 邊界

每條 lane（工作線）判讀時主要看哪一類證據各不相同，但要寫進公開輸出， 全部都得走同一道 Writer 關卡。

Backfill current · 當前回填 writer-gated

回填（backfill）看的不是「這一列有沒有某個標籤」，而是當前這個空白 / 搶救格， 有沒有足夠的「單一樣本」證據、trace、面積、來源追溯與政策支持。

backfill_policy_write_ready_rows

CID-NL Discovery · 發現 identity evidence

Discovery 主要看一列 / 一個 feature 的身分：中性損失標籤、後繼者自證（successor-self）、 MS1 支撐、RT / family 脈絡。候選身分仍要通過啟用合約（activation contract）才能寫入。

CID-NL / MS2 / successor-self evidence

Backfill expansion · 回填擴張 bounded activation

Expansion 不是把 Discovery 候選直接當成矩陣的一列。它必須把每一格的新增壓力， 逐格轉成「單一樣本」證據、trace / 面積值、預期差異與來源追溯。

單一樣本證據 / trace identity / expected-diff

Broad Backfill · 廣域回填 parked 暫緩

Broad Backfill 不靠單一 overlay 或單一分數就啟動。它需要穩定、可重放、可解釋的關卡， 而且要能明確說出哪些格被暫緩或擋下。

broad_backfill_autowrite

Targeted 投影 / 複查動作 expected-diff guarded

Targeted 這條線，由 Product State、Counted Detection、Review State 作為產品判斷。 會改動結果的動作（reject / select / 手動邊界）一律要通過預期差異關卡。

Product State / Counted Detection / Review State

複查 / lockbox / 還原 candidate

這些介面支援人或模型判讀，例如複查佇列、lockbox 盲測/旁證、trace recovery 報告； 它們不是公開輸出 writer 的授權來源。

只做結構化複查 / 只做證據連結還原

05

Common misreads

這些話看起來像結論，其實只是某一層的證據

讀證據鏈的東西時，最容易錯的就是把不同層的字混在一起。 這裡列的是判讀時要避開的「推論跳步」，不是開發任務清單。

一句話原則：任何單一證據來源，只能支持或反駁一段說法； 只有關卡和授權合約，才能決定要不要寫進公開輸出。
Misread 1

有峰，不等於可以寫矩陣

MS1 峰、overlay trace、正面積都只是「訊號 / 數值」證據。它們還要再過身分、 單一樣本歸屬、判定關卡與來源追溯，才靠得近公開輸出 writer。

Misread 2

有 tag，不等於每個樣本都成立

CID-NL / MS2 標籤支持的是「一列」或「候選身分」； 不能直接投影成每個樣本格都有可定量的 MS1 證據。

Misread 3

同一個詞，在不同層意思不同

own_max、ambiguous、rescue、 accepted_rescue、authority 都要標明在哪一層講， 否則會把證據、裁決、書寫範圍混為一談。

Misread 4

教學頁不是現況索引，也不是授權清單

這頁可以教你怎麼看 artifact，但反過來不行——它不能當成啟用數量（activation count）、 書寫範圍、product tier 或矩陣列表的依據。要查現況，以 source-of-truth 文件與 machine-readable artifact 為準。

Misread 5

有些來源只是 hook，還不是完整的書寫線

HCD、標準品、校正、Delta Mass、product-ion 這些語彙已經存在， 但不是每個證據來源都接成一條完整的對外書寫線。報告時要避免過度宣稱。
