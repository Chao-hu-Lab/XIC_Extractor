XIC Extractor product story

從 RAW data 到 final matrix：目前產品主線與下一步優化地圖

這份 report 用白話說明目前 XIC Extractor 如何把 RAW / discovery candidates 變成下游使用的 alignment_matrix.tsv。它同時標出 C4、C6、Region / AsLS 這幾條近期 cleanup 主線已經收斂到哪裡，哪些地方仍然是 bridge、adapter 或 diagnostic。

目前結論

主產品路徑已經不是單一 resolver 或單一 score 決定一切，而是逐步走向 trace、hypothesis、evidence、integration、cross-sample grouping、production decision 的分層架構。

但 C4 舊 scorer、C6 OwnerAlignedFeature delivery adapter、 以及 Region shadow decision 還沒有完全退休或產品化。

Review index 總覽 單樣本 peak C4 evidence Region / baseline C6 cross-sample Final matrix 優化雷達 下一步

一張圖看完主流程

產品主線可以拆成兩層：單樣本層負責從 trace 選出可積分的 peak； cross-sample 層負責判斷哪些 sample-local observations 可以成為同一個 matrix row， 並決定哪些 cell area 可以進 primary matrix。

1. RAW + discovery inputs 讀取 batch index、candidate CSV、RAW paths，建立每個 sample 的輸入集合。
2. XIC trace extraction 對 candidate / backfill seed 回 RAW 拉局部 XIC trace。
3. Peak selection find_peak_and_area 產生 candidates、recovery、scorer selection 與 selected result。
4. Evidence spine 把 selected/candidate facts 投影成 PeakHypothesis、EvidenceVector、IntegrationResult。
5. Cross-sample grouping C6 以 CrossSamplePeakGroupHypothesis 建構，再轉回 delivery DTO。
6. Owner backfill 對缺 sample 的 row 用 group seed 回 RAW 查詢缺失觀察。
7. Cell matrix 建立 detected、rescued、absent、ambiguous、duplicate cells。
8. Production decisions 用 identity、cell quality、claim/consolidation 規則決定 row/cell 是否寫入。
9. Final matrix 輸出 alignment_matrix.tsv 作為 downstream correction/statistics contract。
白話版總結： 現在的產品不是「找到一個峰就直接填 matrix」。它會先在 sample 內選 peak， 再跨 sample 判斷這些 peak 是否屬於同一個 group，接著補缺值、處理重複 claim， 最後才由 production decision 決定哪些值能進 primary matrix。

單樣本層：從 XIC trace 到 selected peak

這一層的產品問題是：在一段 m/z + RT window 裡，到底哪個局部峰應該被積分？ 目前答案仍由 find_peak_and_area(...) 管理。它會先列出 candidates， 再用 RT prior、recovery、scoring context 與可選的 safe merge 決定 selected peak。

現在怎麼做

  • 用 resolver 產生候選 peak。
  • 必要時用 preferred RT recovery 補候選。
  • 有 scoring context 時，使用 legacy scorer 做 candidate selection。
  • region_first_safe_merge 只在明確 resolver mode 下做窄範圍 safe merge。

目前成熟度

產品行為穩定，但語意仍是 bridge：selected peak 的結果已經可以投影到 PeakHypothesis / EvidenceVector，但 successor model 還沒有完全取代 scorer selection。

最重要的風險

如果未來直接刪 scorer 或 region logic，會失去目前仍在保護 selection、 confidence、cap、reason text、boundary behavior 的 active policy。

Surface 角色 目前狀態
xic_extractor/peak_detection/facade.py::find_peak_and_area public production extraction facade，負責 selected peak result。 active policy + bridge
find_peak_candidates(...) 依 resolver_mode 進入 legacy_savgol 或 local_minimum candidate path。 resolver compatibility
select_candidate_with_confidence(...) 目前仍是 scored candidate selection authority。 C4 active policy
build_peak_hypotheses(...) 把 peak result / MS2 evidence / trace facts 投影到 successor audit spine。 successor projection

C4：score 不再是未來語意中心，但 scorer 還沒退休

C4 的方向已經很清楚：未來產品要靠多證據支持或挑戰 peak interpretation， 不是靠單一 raw score 或 cap label 代表真相。這一輪已經建立 EvidenceDecisionSemantics，但 active selection policy 還在 legacy scorer。

重要邊界： EvidenceVector.decision_semantics 現在是 typed projection。 它讓我們可以用 accepted、review、not_counted、 excluded、ambiguous 講同一套語言，但它還不是新的 model-selection engine。
證據家族 目前進 evidence chain 的方式 下一步
MS1 local quality local S/N、trace morphology、shape/width/continuity 目前多半來自 scorer context，再投影成 typed reasons。 把 AsLS baseline provenance、local S/N、shape vector 變成更完整的 trace evidence component。
MS2 / NL candidate-aligned MS2 / neutral loss 會成為 support 或 conflict；missing MS2 預設不是硬負證據。 持續要求 acquisition opportunity / trigger context，避免 missing evidence 被誤用。
RT / ISTD / STD RT 是 contextual evidence。ISTD / STD 配對可提供 role-aware RT support 或 review reason。 把 ISTD real-matrix transfer evidence 與 clean standard context 分清楚。
CWT / boundary morphology CWT 可提供 morphology / boundary-hypothesis context；不是單獨決定 identity。 不要說成只能 support。正確說法是任何單一證據都不能單獨定奪。
工程細節
  • xic_extractor/evidence_semantics.py 定義 CommonEvidence、EvidenceSignalSet、EvidenceDecisionSemantics。
  • decision_semantics_from_signal_set(...) 把 support / concern / cap / quality labels 轉成 typed decision classes and reasons。
  • xic_extractor/peak_detection/hypotheses.py::EvidenceVector 已加入 confidence、cap_labels、reason、decision_semantics。
  • score_candidate(...)、select_candidate_with_confidence(...)、confidence thresholds、caps、reason text 仍是 active product policy。

Region / baseline：AsLS 是定量基礎，boundary 還是下一個可優化點

定量問題的核心已經先從 baseline 開始修正：linear_edge 不再是產品 baseline integration method， asls 是目前唯一允許的 baseline method。Region boundary 則是另一個問題： 它不是死碼，但也還不是完整產品 selector。

Baseline 現況

integrate_with_baseline(...) 只接受 asls。 如果傳入 linear_edge 會丟出退休訊息。這代表後續不應再比較 linear edge 是否能回到產品路徑，而是要驗證 AsLS area、uncertainty、baseline residual 如何投影到 matrix 與 audit。

Boundary 現況

RegionSelectionDecision 已經有產品/影子共用的 decision language。 但 shadow region model-selection 仍主要是 diagnostic/review；只有 region_first_safe_merge 的窄 gate 會影響產品 selected interval。

白話版： baseline 是「面積怎麼算才不低估」；boundary 是「哪一段 trace 應該被拿去積分」。 AsLS 已經先解決 baseline 方向，但 boundary/model-selection 還有精進空間。
工程細節
Surface 角色 狀態
peak_detection/baseline.py AsLS baseline、baseline-corrected area、uncertainty/residual MAD。 AsLS active
LINEAR_EDGE_RETIRED_MESSAGE 防止 retired baseline 被重新接回產品或 audit path。 retirement guard
peak_detection/region_model_selection.py 把 boundary / WIS / CWT 等 evidence 變成 region decision。 diagnostic owner
peak_detection/region_safe_merge.py 只在 safe adjacent-WIS merge gate 成立時影響產品 interval。 narrow product gate

C6：family 被降級，cross-sample group hypothesis 成為 constructor

C6 的核心不是「刪掉 family」而是停止把 family 當成真相。 現在比較正確的語意是：多個 sample-local peak observations 可能代表同一個 cross-sample peak group，這個 claim 可以被支持、挑戰、split、demote。

Sample-local owners 每個 sample 裡由 MS1/MS2 candidate 形成 owner observation。
CrossSamplePeakGroupHypothesis 新的 constructor 語意，承載 group membership claim。
OwnerAlignedFeature 舊 public delivery DTO，仍餵給 backfill/matrix/writers。
AlignmentMatrix 下游仍用 family-compatible row ID 寫出 matrix。
目前定位： C6 已經不是 event-first 老系統；也不是完全完成 successor migration。 它現在是「successor constructor + legacy delivery adapter」。
工程細節
  • alignment/cross_sample_peak_groups.py::CrossSamplePeakGroupHypothesis 是目前 C6 successor constructor model。
  • alignment/owner_clustering.py::cluster_sample_local_owners 會先建 hypothesis，再轉成 OwnerAlignedFeature。
  • OwnerAlignedFeature 仍是 backfill、matrix、claim registry、primary consolidation、writer、process payload 的 public delivery language。
  • FAM###### 仍是 output compatibility row ID，不是 domain truth。

Final matrix：不是所有 cell 都會寫入 primary matrix

final matrix 的重點是 alignment_matrix.tsv。它是 downstream correction/statistics delivery surface。現在 writer 不會把所有 AlignedCell.area 直接倒進去， 而是先建立 production decision set。

步驟 做什麼 產品意義
Owner backfill 用 group seed 回 RAW 查缺 sample 的 XIC，產生 rescued cell。 補上「有可能存在但 discovery 沒抓到」的觀察，但不代表一定可進 matrix。
Claim registry 處理同一 sample 裡多個 row claim 到同一 MS1 peak 的情況。 避免同一個實際 peak 被多個 row 重複定量。
Primary consolidation 合併或降級 near-duplicate family competitors。 把 row-level identity 壓回較一致的 primary matrix surface。
Production decision 用 cell quality + matrix identity 決定 row/cell 是否寫值。 final matrix 只寫 accepted production values，review/ambiguous/duplicate/absent 會留空或轉 review surface。

Matrix value source

AlignedCell.matrix_area 會優先使用 selected_integration.area_raw_counts_seconds， 沒有 selected integration 才走 legacy fallback。這是 handoff spine 已經 production-ready 的部分。

仍要注意

linear edge 已退休，所以 final matrix 不應再維持 linear-edge 或 legacy-baseline 主值語意。下一步要定義的是 AsLS baseline-corrected area 如何成為 primary matrix value，以及 uncertainty / rollback audit 如何被清楚命名。

工程細節
  • alignment/owner_matrix.py::build_owner_alignment_matrix 建立 detected/rescued/absent/ambiguous cells。
  • alignment/claim_registry.py::apply_ms1_peak_claim_registry 處理 duplicate MS1 peak claim。
  • alignment/primary_consolidation.py::consolidate_primary_family_rows 處理 primary row consolidation。
  • alignment/production_decisions.py::build_production_decisions 是 matrix writer 前的 row/cell decision layer。
  • alignment/tsv_writer.py::write_alignment_matrix_tsv 寫出 production_matrix_area(decision)。

優化雷達：目前最像雙軌維護的地方

這裡不是說每一項都要馬上刪，而是標出「舊語意已被新語意吸收一部分， 但產品行為或 delivery surface 還沒有完全遷移」的位置。

熱區 現在狀態 為什麼還不能直接刪 合理 exit rule
C4 scorer policy bridge evidence semantics 已建立，但 scorer 還是 selection/confidence/caps authority。 直接刪會改 selected candidate、reason text、public projection。 建立 successor model-selection parity，再把 scorer 縮成 compatibility projection 或退休。
Region boundary diagnostic + narrow gate shadow decision 可解釋問題，safe merge 只接窄路徑。 boundary model-selection 還沒成為完整 product selector。 定義 promotion/kill/externalize 規則，把 region facts 融入 hypothesis/integration/audit spine。
C6 owner delivery successor constructor + delivery adapter group hypothesis 已建構，但下游還吃 DTO。 backfill/matrix/claim/writer/process payload 還依賴 OwnerAlignedFeature shape。 把 downstream contract 改吃 owner-group delivery protocol 或 successor projection，再評估 DTO 退休。
Matrix area semantics product decision linear edge 已退休；final matrix 主值必須收斂到 AsLS corrected area policy。 直接換 matrix value 會影響所有下游定量結果，但不能因此保留 retired baseline 主值。 寫 AsLS primary matrix policy：AsLS 作主值，歷史值只能是明確命名的 audit/rollback。

下一輪方向建議

如果目標是「用一條主線把產品語意收乾淨」，我建議不要先追所有死碼。 先處理會改變產品理解與維護成本的三個 decision。

定量 area policy：AsLS corrected area 在 final matrix 的角色

baseline 已經改成 AsLS，final matrix 不應再沿用 retired baseline 主值。 這一步要定義 AsLS-corrected area 作為 primary value，以及 uncertainty / audit projection。

接著做

Region boundary model-selection：從 diagnostic 走到可驗證 promotion

OpenMS-style thinking 的重點不是選某個 resolver 名稱，而是把 peak 選對、boundary 選對、 evidence 給足。下一步應該設計最小 promotion gate，而不是再讓 region shadow 永久旁觀。

3
結構收斂

C6 downstream migration：讓 hypothesis/group contract 真正餵 matrix

C6 已經把 successor constructor 放進來，下一步不是再討論 family 叫什麼， 而是讓 backfill、claim、matrix、writer 逐步依賴 successor contract 或 protocol。

adapter cleanup
4
後續

C4 successor selection：把 scorer 從 active policy 降級

這會比 C6 adapter cleanup 更容易碰到行為改變，應該在 typed evidence parity 和 selected hypothesis oracle 足夠後再動。

later migration
建議故事線： 先把定量主值講清楚，因為 final matrix 最終是定量產品； 再處理 boundary，因為 boundary 會影響 area 是否可信； 接著把 C6 downstream adapter 收斂，降低維護兩套 cross-sample 語意的成本； 最後再碰 C4 model selection，因為它最容易牽動 public reason/confidence/selection 行為。
