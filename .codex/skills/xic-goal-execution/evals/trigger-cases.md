# Trigger Cases

## Positive Triggers

- "幫我把這個 XIC phase plan 變成可以 end-to-end 執行的 /goal。"
- "繼續執行 XIC goal，直到 8RAW gate 通過或 stop rule 觸發。"
- "Review this XIC goal contract before we run the validation."
- "這個 productization task 一直 drift，幫我收成單一 finish line。"

## Negative Triggers

- "跑一下 git status。"
- "修一個 README typo。"
- "只跑既有 one-RAW command，acceptance surface 已經很清楚。"
- "先 brainstorming product direction，不要開始執行。"

## Near Neighbors

- Use `xic-raw-validation` for focused RAW launch/acceptance with an obvious
  done state.
- Use `xic-pr-closeout` for PR/branch narrative and readiness closeout.
- Use `xic-architecture-preflight` before non-trivial architecture or
  evidence-provider implementation.
