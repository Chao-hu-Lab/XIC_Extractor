# Trigger Cases

## Positive Triggers

- "Review this large XIC PR for matrix identity and activation risks."
- "Do a read-only review of this RAW locality PR."
- "This diagnostics-heavy branch needs blast-radius review."
- "Check whether this 8RAW parity PR overclaims 85RAW readiness."

## Negative Triggers

- "Prepare the PR description."
- "Merge this PR."
- "Fix the review comments."
- "Review this tiny one-file typo patch."

## Near Neighbors

- Use `xic-pr-closeout` for PR narrative, verification summary, or closeout.
- Use GitHub review-comment skills for addressing requested changes.
- Use `critical-artifact-review` for specs/plans rather than code diffs.
