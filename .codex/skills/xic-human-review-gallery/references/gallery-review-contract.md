# Gallery Review Contract

## Owner Reuse

Before adding a new HTML/report:

- search existing Gallery, overlay, guide, browser-smoke, and retention helpers;
- deepen the existing owner when it fits;
- state why a new sibling artifact is necessary when reuse does not fit.

Do not create a new Gallery system just because the current surface is hard to
read.

## Review Lens

Use one primary lens per artifact:

- Backfill context: does current row/sample decision have reasonable MS1 context?
- Discovery identity: which row identity better explains the same samples?
- Authority boundary: what can enter review versus product authority?
- Tutorial/guide: how should a human read the evidence surfaces?

Avoid text that asks "should we backfill?" while the figure is really testing
Discovery, or vice versa.

## Visual Rules

- Put explanation next to the visual element it describes.
- Do not rely on tiny floating pins that detach under browser zoom or comments.
- Use clean generated or real images when a tutorial needs visual teaching.
- Use in-app Browser for local smoke unless a task specifically requires Chrome
  state, cookies, extensions, or logged-in sessions.
- Check desktop and narrow/mobile layouts for overlap and horizontal overflow.

## Authority Boundary

Gallery and overlay surfaces can support review conclusions. They do not by
themselves create ProductWriter authority, default matrix writes, or activation
policy changes.
