# Overlay Verification

Use this after generating an overlay image, embedded HTML figure, or tutorial
graphic.

## Image Checks

- nonblank image;
- expected traces or panels present;
- legend/labels match plotted trace count;
- primary trace roles remain distinguishable under overlap;
- axes or RT window are visible when relevant;
- smoothing/normalization is disclosed;
- annotations do not cover key evidence;
- default Backfill review overlays do not include derived median/consensus
  traces as if they were sample evidence;
- source/successor/differential language matches the plotted identity.

## Browser / HTML Checks

When the overlay is embedded in HTML:

- use in-app Browser for local pages unless Chrome state is explicitly required;
- inspect desktop and narrow/mobile widths;
- check no horizontal overflow;
- check sticky headers, comments, or nav do not cover the target section;
- check image dimensions and object-fit do not crop the evidence;
- check captions and explanation are adjacent to the visual they explain.

## Retention Checks

- Keep generated PNG bundles, trace JSON, full TSVs, and gallery outputs
  externalized unless an approved public contract says otherwise.
- Track compact guide/index/summary files when they are durable review surfaces.
- Update artifact inventory or retention checker when tracked guide/index files
  change.

## Calibration Record

For first real use, record:

- command or script that generated the overlay;
- data/artifact source;
- checks passed and failed;
- human readability issues;
- plotting owner or skill rule that should be tightened next.
