# Trigger Cases

## Positive Triggers

- "Before coding this CID-NL evidence provider, write the architecture preflight."
- "This diagnostic needs RAW-backed evidence and may affect matrix authority."
- "Plan the preset-performance change and call-cost model first."
- "We need to add a Delta Mass lane without creating another owner."

## Negative Triggers

- "Fix a typo in docs."
- "Run an existing focused unit test."
- "Review a completed PR without editing code."
- "Read an artifact summary for status only."

## Near Neighbors

- Use `xic-raw-validation` for RAW launch/acceptance gates.
- Use `xic-large-pr-review` for read-only PR review.
- Use `xic-product-gate-advancement` when deciding whether evidence promotes a
  product gate.
